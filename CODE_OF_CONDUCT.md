# Code of Conduct

OpenCircuitModel is a small project run by humans. This document is short on purpose.

## TL;DR

**Be honest, be kind, assume good intent.** Disagree with ideas, not people. If a thread gets unproductive, take a day. The project is more important than any single dispute.

## Expected behavior

- **Be respectful.** Even when you think someone is wrong. Especially then.
- **Be specific.** "This benchmark is misleading because it elides the cache state" is useful. "This benchmark is bad" is not.
- **Be patient with newcomers.** OCM stands on a tall stack of borrowed frameworks; expecting fluency in all of them on day one is unfair.
- **Credit upstream.** When your contribution touches a borrowed framework's surface area, name the upstream in the commit body.
- **Keep technical disagreements technical.** Bring data; rebut data with data.

## Unacceptable behavior

- Personal attacks, slurs, harassment, or sustained disruption — public or private. Reported instances result in immediate removal from the project.
- Sharing other contributors' private information (real name if they use a handle, email, location, employer) without explicit permission.
- Bad-faith engagement: deliberate misreading, sealioning, drive-by criticism without engagement.
- Commercial spam, recruitment spam, off-topic promotion in PRs / issues.

## Enforcement

The project lead (Brand, becky@nativeteachingaids.com) makes enforcement decisions for now. As the contributor base grows, this gets a small enforcement team with documented escalation steps.

Decisions can be:
- A private warning explaining what crossed a line
- A public correction in the offending thread
- A temporary ban from project communication channels
- A permanent ban for repeated or severe violations

## How to report

Email **becky@nativeteachingaids.com** with:
- What happened (link to thread / commit / message if possible)
- Who was involved
- Whether you want to be named or kept anonymous in any follow-up

Reports are confidential. We respond within 72 hours of receipt.
